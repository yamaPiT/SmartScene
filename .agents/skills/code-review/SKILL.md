# コードレビュー・スキル (Code Review Skill)

コードをレビューする際は、以下のステップに従って厳格にチェックを行ってください。

## レビューのチェックリスト
1. **正確性 (Correctness)**: コードは要求仕様書等の意図通りに動作するか？
2. **エッジケース (Edge cases)**: エラー状態や想定外の入力が適切にハンドリングされているか？
3. **スタイル (Style)**: プロジェクトのコーディング規約やASDoQ品質モデルに準拠しているか？
4. **パフォーマンス (Performance)**: 明らかな非効率性やボトルネックが存在しないか？

## フィードバックの提供方法
- 何を変更する必要があるかを「具体的」に指摘すること。
- 何（What）を変更するかだけでなく、なぜ（Why）それが必要かを説明すること。
- 可能な限り、具体的な代替案やコードサンプルを提示すること。
