# OSDVIスマートシナリオプロトタイプシステム (Smart Scene Simulator)

本プロジェクトは、車載ソフトウェアのアーキテクチャ、ECUの状態遷移、およびHMI（ヒューマン・マシン・インターフェース）の設計原則を実践的に学ぶためのWebベースの教育用シミュレータです。

ユーザーは、ダッシュボードの操作を通じて「マニュアル操作と自動制御（スマート機能）の境界」を体験し、自然言語による要求仕様がどのように標準API（OSDVI / VSS準拠）へマッピングされるかを学習できます。

## 🌟 ADA (Agent-Driven Agile) 開発プロセスの実践と再利用

本システムの開発を通じて、AIエージェント（Antigravity）とのペアプログラミングにおける開発プロセスを体系化し、これを**「ADA (Agent-Driven Agile)」**と名付けました。本プロジェクトには、AIを単なるコーディングアシスタントとしてではなく、要件定義からテスト・レビューまでの全工程をともに担う「優れたパートナー」として機能させるための仕組みが組み込まれています。

開発の実践を通して、Antigravityのための**ルール、スキル、ワークフローを整備**し、ADA開発プロセスに準拠して自動的にV字モデルを遂行できるようにしました。なお、現在のADA開発プロセスはまだ初期のプロトタイプ段階（未完成）であり、今後、他のプロジェクトでの開発経験を積み重ねながら改定・アップデートを行っていく予定です。

以下のフォルダ群を他のプロジェクトにコピーして再利用することで、ユーザーは全く新しいプロジェクトでも簡単にADA開発プロセスの仕様・恩恵を受けることができます。

*   **`.cursor/rules/` (Cursor/Antigravityルール)**:
    *   AIへの前提条件や絶対的なプロジェクトルール（MDC形式）が格納されています。コード先行の禁止や、人間（Product Owner）とAIエージェントの明確な役割分担が定義されています。
*   **`.agents/` (Antigravity設定)**: 
    *   **ワークフロー (`workflows/`)**: ADAプロセスのV字モデル標準開発手順が定義されています。チャットで `/ADA-Process` と呼ぶことで発動します。
    *   **スキル (`skills/`)**: V字モデルの各工程を担当する専門エージェント（要件、設計、実装、テスト）と、ASDoQ文書品質モデルに基づき品質を保証する高次レビュアーエージェントのスキル（計8つ）が含まれています。
*   **`docs/` (ドキュメントと参照資料)**: 
    *   ASDoQ文書品質モデルなど、開発およびドキュメント作成時にAIと人間が参照すべき品質基準となる共通資料が配置されています。

## ✨ シミュレータの主な機能と学習ポイント

* **イグニッション状態遷移**: START/STOPによるシステム全体（アクチュエータやシナリオエンジン）の稼働統制。
* **物理挙動のシミュレーション**: 窓の開閉モーター制御（3秒間での目標値追従）や、長押しによるマニュアル/オート開閉の切り替え。
* **高度なオーバーライド（手動介入）ロジック**: 自動制御中にドライバーが手動操作を行った際、システムが優先権を譲る「ドライバー・イン・ザ・ループ」の原則の実装。
* **エッジ検出によるシナリオ復帰**: 手動介入後も、環境条件（雨量など）が再成立した瞬間に自動制御へシームレスに復帰する最新の制御ロジック。
* **教育用シナリオ・ビューア**: 要件定義（自然言語）と実装コード（OSDVI API）を併記し、Read-Onlyで表示。

## 🚀 搭載されているスマートシーン

1. **雨天時スマートシーン**: 雨量センサ（0〜100%）に連動し、窓の自動閉鎖、ワイパー動作、デフロスタ/デフォッガの自動起動を行います。雨が止むと元の状態に復元（RESTORE）します。
2. **サンキューハザード**: 走行中にウインカーを操作すると、レーンチェンジ後に自動でハザードが3秒間点滅し、周囲に感謝を伝えるシーケンスが発動します。

### 💡 スマートシーンの開発経緯と仕様書の連携について

* 搭載されているスマートシーンを作成する際に、例えば降雨量センサの名称などを設定するときには、OSDVIが [https://www.nces.i.nagoya-u.ac.jp/osdvi/index.html](https://www.nces.i.nagoya-u.ac.jp/osdvi/index.html) で公開している「Open SDV API バージョン202509α 発行：2025年9月30日」を参照しました。
* 実際には、それらの文書をAI（Gemini Web）に読み込ませた上で、要求仕様書と設計書を作成しました。要求仕様書と設計書を作成したAIは、Webブラウザから起動するGeminiであり、Antigravityのチャット欄のAIではありません。
* その理由は、開発当初にはAntigravityの使い方がよく分からず、AntigravityのAIに仕様書を読み込ませる方法が分からなかったからです。
* 今後は、`./doc` フォルダ下に仕様書を置いて、それをAIに参照させる指示を出せば、Antigravity内のAIで、新しいスマートシーンシナリオを作成できる可能性があります。
* なお、AIが仕様書から名前を引用しており、引用元の仕様書はルールエディタのルールにコメント記述されています。

## 📋 前提環境とインストール要件

本システムの開発および実行には以下の環境のインストールが必要です。

* **Node.js**: アプリケーションの動作に必要です。[Node.js 公式サイト](https://nodejs.org/)からインストーラをダウンロードし、ご自身の環境にインストールしてください。
* **npm (Node Package Manager)**: Node.jsに同梱されていますが、バージョンが古い場合やバックグラウンドでの非同期なアップデートによって、インストール時に警告やエラーが生じることがあります。ユーザーの利便性およびトラブル防止のため、事前にターミナルで `npm install -g npm@latest` を実行し、npmを最新バージョンへアップデートしておくことを推奨します。

**【npmの脆弱性診断と対応（推奨）】**
npmパッケージは日々アップデートされており、使用しているライブラリに脆弱性が発見されることがあります。セキュリティを保つため、定期的にターミナルで `npm audit` を実行して脆弱性をチェックし、問題が見つかった場合は `npm audit fix` を実行して安全なバージョンへ自動的にアップデート・修正を行うことをお勧めします。

**【3D表示用ライブラリについて】**
カーシミュレータの3Dモデル構築および表示には、`three`, `@react-three/fiber`, `@react-three/drei` などのライブラリを使用しています。これらは後述の起動手順に記載されている `npm install` コマンドを実行した際に一括で自動インストールされるため、ご自身で個別にインストールコマンドを実行する必要はありません。

## 💻 起動方法１：Google Antigravity をお使いの方（推奨）

Google AntigravityがインストールされているPCであれば、最も簡単に環境を構築できます。
※他者のリポジトリをダウンロードして動かすだけであれば、**GitHubのアカウントは不要**です。（ご自身でコードを改変してGitHubに保存し直したい場合のみ、アカウント作成とログインが必要です）。

**1. リポジトリの読み込み（Clone）**
* ブラウザでこのGitHubページを開き、上部にある緑色の `Code` ボタンを押し、表示されるURL（`https://github.com/...`）をコピーします。
* Antigravityを起動します。起動すると「Antigravity」「Agent Manager」「Editor」という3つのウィンドウが開きますが、ここでは**「Editor」ウィンドウ**を使用します。
* Editorウィンドウの左側にある**「ソース管理（Git）アイコン」**（枝分かれしたようなマーク）をクリックします。
* 表示されたサイドバーの中から**「リポジトリをクローンする (Clone Repository)」**ボタンを押下します。
* 画面上部に入力欄が表示されるので、先ほどコピーしたURLを貼り付けてEnterキーを押し、保存先のフォルダを指定するとファイル一式が読み込まれます。

**2. 起動コマンドの実行**
* Editorウィンドウの上部メニューから `Terminal`（ターミナル）→ `New Terminal`（新しいターミナル）を開きます。
* 画面下部に開いたターミナル（黒い画面）に以下のコマンドを1行ずつ入力し、Enterキーを押します。

```bash
npm install
npm run dev
```

**3. シミュレータの表示**
* 起動に成功すると、Editorウィンドウ右側の「Browser（ブラウザ）」タブ、またはご自身のWebブラウザ（http://localhost:3000）でシミュレータが操作可能になります。

## 💻 起動方法２：通常のパソコン（ローカル環境）でお使いの方

Antigravityをお持ちでない方や、プログラミング初心者の方は以下の手順で動かすことができます。

**1. プログラムのダウンロードと解凍**
* このGitHub画面の右上にある緑色の `Code` ボタンをクリックし、`Download ZIP` を選んでパソコンに保存します。
* 保存したZIPファイルを右クリックし、「すべて展開」などでフォルダとして解凍してください。

**2. コマンドライン（ターミナル）の準備**
* 解凍したフォルダを開き、その場所で「ターミナル（Mac）」または「コマンドプロンプト / PowerShell（Windows）」を開きます。
* ※VS Codeをお使いの場合は、解凍したフォルダをVS Codeで開き、上部メニューの「ターミナル」→「新しいターミナル」を開いてください。

**3. 起動コマンドの実行**
* 以下のコマンドを1行ずつ入力し、Enterキーを押して実行します。

```bash
npm install
npm run dev
```

**4. ブラウザで確認**
* コマンドの実行が終わったら、お使いのブラウザ（ChromeやEdgeなど）のアドレスバーに `http://localhost:3000` と入力してください。シミュレータが起動します。

## 🤖 詳細：AIエージェントとの協調開発（ADA V字モデルプロセス）

本プロジェクト（または、ここから各フォルダをコピーした開発環境）では、コードの品質と設計の整合性を保つため、体系化された「ADAプロセス（V字モデル化）」に則って作業を進める仕組みがあります。

> 📖 **詳細な学習教材**  
> AIエージェントとの実践的なペアプログラミング手法のより深い解説は、**[AIエージェントとの協調開発ガイド (docs/agent_development_guide.md)](docs/agent_development_guide.md)** をご覧ください。

### 具体的な開発手順（ADAプロセスの利用）
Antigravityのチャットウィンドウで機能の追加や修正を依頼する際は、必ず**文の先頭に `/ADA-Process` と入力**してください。

**入力例：**
> `/ADA-Process 〇〇機能を追加してください。`

このコマンドを入力することで、AI（ハル）が以下の「ADAプロセス（V字モデル標準プロセス）」を自動的に遵守して作業を進めます。各工程には専門のエージェントスキルが割り当てられ、作成とASDoQ準拠の厳格なレビューがセットで行われます。

1. **要求定義フェーズ**: `requirements-engineer`による要件定義と、`requirements-reviewer`による目的適合性のレビュー。
2. **アーキテクチャ設計フェーズ**: `architect`による設計と、`architecture-reviewer`による保守性・整合性のレビュー。
3. **実装フェーズ**: `programmer`によるコード実装と、`code-review`による品質・保守性のレビュー。
4. **テスト・検証フェーズ**: `test-engineer`によるテストケース構築・実行と、`test-reviewer`によるテスト品質のレビュー。

※全工程を通じて、ドキュメントのASDoQ準拠を保証するための共通スキル `asdoq-compliance` が活用されます。

### 🌟 搭載されているエージェントスキルとワークフロー
本プロジェクトでは開発の各フェーズを専門的にサポートするため、以下のスキル群とワークフローが組み込まれています。チャットウィンドウから個別に呼び出して活用することも可能です。

#### ワークフロー (`.agents/workflows/`)
* **`/ADA-Process`**: ADA V字モデル標準開発プロセスを起動します。
* **`/generate-unit-tests`**: 実装した機能に対するユニットテストを自律的に生成します。

#### エージェントスキル (`.agents/skills/`)
* **要求フェーズ**: `requirements-engineer` (作成), `requirements-reviewer` (レビュー)
* **設計フェーズ**: `architect` (作成), `architecture-reviewer` (レビュー)
* **実装フェーズ**: `programmer` (実装), `code-review` (レビュー)
* **テストフェーズ**: `test-engineer` (作成・実行), `test-reviewer` (レビュー)
* **共通スキル**: `asdoq-compliance` (ASDoQ文書品質基準の適合チェック)

## 📁 主要ファイルとルールの設定変更

本プロジェクトには、AIエージェントの振る舞いを定義する各種ファイルが含まれており、GitHubからプルしてエディタを開いた時点で**自動的に環境に適応**されます。

* **ワークフロー**: `.agents/workflows/`
* **スキル**: `.agents/skills/`
* **基本ルール**: `.cursor/rules/project-rules.mdc` (AIの絶対ルール・MDC形式)

### 💡 ペルソナ（登場人物）のカスタマイズについて
AIが必ず日本語で回答することや、常にADAプロセス（プロジェクトルール・開発文書）に従って開発を進めることなどの基本ルールは、新たに設けられた `.cursor/rules/project-rules.mdc` に定義されています。Antigravityなどの互換エディタは自動的にこのMDC（Markdown Cursor）形式のルールを参照し続けます。

デフォルトでは、教育目的のロールプレイとして**「ユーザー名：マサ、AI名：ハル」**という個人的なペルソナが固定で設定されています。

このリポジトリを使う他者（ご自身）の名前や好みのAI名で開発を進めたい場合は、エディタで `.cursor/rules/project-rules.mdc` を開き、以下の該当箇所をご自身の名称に書き換えてください。

```markdown
私の名前は[あなたの名前]（Product Owner）です。あなたは優秀なテクニカルパートナーの「[好きなAIの名前]」です。
[あなたの名前]に対しては技術的なパートナーとして最大限の敬意を持ちつつ...
```

## � context7 (MCPサーバー) の設定について

このリポジトリでは、`context7` というAIがライブラリを使うときに最新のドキュメントを参照できるようにするMCPサーバーを用いています。具体的には、次の設定をしています。設定が難しい場合は、`.\.cursor\rules\use-context7-for-docs.mdc` ファイルを削除してください。

### (1) context7 API Keyの取得
* [https://context7.com/](https://context7.com/) にGoogleアカウントなどでサインインする。
* 上部の `More...` メニュー内に `Create API Key` があるので、そこでAPI Keyを作成してコピーする。

### (2) AntigravityでのMCPサーバー設定
* Antigravity画面右上の三点ドットから `MCP Servers` を選択し、`View raw config` を選択する。
* 以下のように作成または追記する。`YOUR_API_KEY` に、先ほど取得したキーを入力する。

```json
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp", "--api-key", "YOUR_API_KEY"]
    }
  }
}
```
