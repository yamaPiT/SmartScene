1. 概要

本ソフトウェアの存在意義と、プロジェクト全体における前提を定義する。

1.1 目的と位置づけ

本ソフトウェアは、「OSDVIスマートシーンデモシステム V2.0」における要求仕様を定義する。
ユーザーがダッシュボードの操作を通じて「マニュアル操作と自動制御の境界」を体験し、最新の標準API（OSDVI 202603α）の動作を、視覚的にリッチな3D環境で確認・体感可能とすることを目的とする。

1.2 適用範囲

Webベースの車載ソフトウェアデモシミュレータに適用する。

1.3 用語定義

OSDVI: 自動車のソフトウェア定義（SDV）における標準API仕様（最新: 202603α版）。

DADAプロセス: ドキュメント駆動によるAI協調開発プロセス。

Zustand: 本システムの状態管理（OSDVIモックストア）に使用するライブラリ。

2. 視覚的・体験的要求（既存資産の継承）

本システムは、機能的な正確さだけでなく、以下の3Dアニメーション表現を必須要件として継続・維持する。

[REQ-V01] 3D車両モデルのリアルタイム表示

内容: ブラウザ上に車両の3Dモデルを常時表示し、全方位からの視点変更を可能とする。

継承事項: 既存のWebGL/Three.jsベースのレンダリングエンジンを維持すること。

[REQ-V02] 天候アニメーション（雨）

内容: Vehicle.Exterior.Air.RainIntensity の値に連動して、画面内の降雨アニメーションの密度と強弱がリアルタイムに変化すること。

[REQ-V03] アクチュエータ動作アニメーション（窓）

内容: Vehicle.Cabin.Window.$Instance.Position の変化に伴い、3Dモデル上の窓がスムーズに開閉（昇降）すること。0%（全閉）から100%（全開）までの連続的なアニメーションを維持する。

[REQ-V04] デフォッガ動作視覚効果

内容: デフォッガ（Defogger）がONの状態の際、窓ガラスの曇りが消えていく、または動作中を示すエフェクトを表示すること。

3. 機能要求

[REQ-F01] OSDVI 202603α準拠の状態管理

システム内部のデータ構造およびAPIインターフェースは、後述の第6章「OSDVI API実装ガイド」に完全に準拠すること。

[REQ-F02] スマートシナリオによる自動制御

降雨検知時の窓の自動閉鎖や、特定の条件下でのデフォッガ起動など、定義されたシナリオに基づくサービスコール発行を行うこと。

[REQ-F03] 手動操作との調停（後勝ち原則）と状態の原状復帰

シナリオ稼働中（雨天時など）にユーザーによる手動操作があった場合、最新の要求を優先して反映すること（マニュアルオーバーライド）。ただし、シナリオ終了条件（雨上がり等）が成立した際は、シナリオ中の手動操作の有無に関わらず、対象となる機能要素（窓、デフォッガ、ワイパー等）を「シナリオが発動する直前の状態」へ強制的に原状復帰させること。

4. 非機能要求

パフォーマンス: 3Dアニメーションの描画とAPI状態の更新を、ユーザーに遅延を感じさせない速度（目標60fps）で行うこと。

エラー耐性: 不正なAPIパラメータが入力された場合でも、システムがクラッシュせず、OSDVI定義のエラーコード（E_INVALID_PARAMETER等）を返すこと。

5. 外部インタフェース要求

操作パネル: ユーザーが車両の状態（イグニッション、窓、ワイパー等）を操作できるスイッチ群。

APIビューア: 現在発行されているOSDVI APIのサービスコール名や引数をリアルタイムにコードスニペットとして表示する機能。

6. OSDVI API (202603α版) 実装ガイド

Antigravityが設計・実装時に参照すべき「唯一の情報源」として、最新仕様を以下に定義する。

6.1 基本規約

ネーミングルール: 全ての列挙型（enum）およびその列挙子は PascalCase（先頭大文字キャメルケース） とする。

インスタンス識別: マルチインスタンス（右窓・左窓等）のプレフィックスには $ を使用する（例: $FrontRight, $FrontLeft）。

6.2 Vehicle.Motion（車両運動）

応答プロファイル（ResponseProfile）:
以下の4種に統合された。これら以外の名称は使用しない。

Standard: 標準的な加減速

Maximum: 最大性能

Rapid: 急な加減速

Gentle: 緩やかな加減速

リスククラス: RiskVehicleInstability（車両不安定リスク）が追加された。

6.3 Vehicle.Cabin.Window（ウィンドウ制御）

主要サービスコール:

startMove(position, priority): 目標位置（0-100）と優先度を指定。

stopMove(priority): 現在位置で停止。

lock(priority) / unlock(): 操作権限のロックと解除。

イベント種別 (WindowEventKindType):
状態変化時に以下のイベントを発行すること。

ControlledBySelf, ControlledByOther: 制御主体の識別。

TargetReached: 目標到達。

PinchDetected: 挟み込み検知（検知時は即座に停止し、反転動作へ移行すること）。

OwnLockRevoked: ロックの強制解除。

6.4 Vehicle.Exterior.Light.Defogger（デフォッガ）

状態管理: IsActive (Boolean) および Mode (Enum) によって管理する。
