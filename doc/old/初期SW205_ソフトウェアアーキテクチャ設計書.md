# ソフトウェアアーキテクチャ設計書

## 1. アーキテクチャ概要
* **パターン**: コンポーネント指向アーキテクチャ (React) + 状態駆動ループ
* **データフロー**: 単方向データフロー。UIはRead-Onlyなシナリオを表示し、エグゼキュータが100ms周期で状態を評価・更新する。

## 2. データ構造設計 (Vehicle State)
* **ステート管理**: アプリケーションの全体状態はZustandを用いた一元的なStore(`useVehicleStore`)で管理する。
* **VSS準拠キー**: 車両の各センサーおよびアクチュエータの状態は、OSDVI/VSS規格に準拠したドット区切りの名前空間（例：`Vehicle.Speed`, `Vehicle.Exterior.Air.RainIntensity`, `Vehicle.Cabin.Door.Row1.Left.Window.Position`等）をキーとして保持する。
* **内部拡張キー**: システム特有の制御状態（イグニッション状態、FanSpeed、オーバーライドフラグ、過去の状態キャッシュ、シナリオの実行状況など）は `Internal.*` プレフィックス等を付与して厳密に状態構造を維持する。

## 3. サブシステム設計

### 3.1 クルマシミュレータ (Vehicle Simulator)
* **イグニッション制御 (START/STOPボタン)**:
  * コンテナのフレックスボックス等のレイアウト設定を調整し、円形のSTART/STOPボタンをエリアの左端（`justify-start` 等）に配置する。
* **窓ボタンの手動制御アーキテクチャ (Press and Hold)**:
  * 窓のOP/CLボタンは、クリック（`onClick`）ではなく、マウスダウン (`onMouseDown`, `onTouchStart`) およびマウスアップ (`onMouseUp`, `onMouseLeave`, `onTouchEnd`) イベントを利用して制御する。
  * マウスダウン時に、対象窓に対して `OP` (開く方向へ全開速度で駆動) または `CL` (閉じる方向へ全閉速度で駆動) の開始シグナルを送信する。
  * マウスアップ（または領域外へのカーソル移動等）時に、対象窓に対して動作停止（Stop）シグナルを送信し、その時点の開度を維持させる。
* **窓の開度テキストと中央ステータス表示（視認性向上）**:
  * 窓の開度（Front/Rear: X%）および中央のステータス（`LANE CHANGE` / `♥THANKYU♥`）を表示するコンテナのCSS背景色について、`bg-opacity` などの半透明指定を完全に排除し、`bg-black` などの不透明（Opacity 1.0）な単色塗りつぶしを適用する。これにより、背後の雨量パーティクルによる視認性低下を物理的に防ぐ。
* **ワイパー描画の幾何学的補正**:
  * SVGまたはCSSによるワイパーアーム（直線要素）の描画において、座標系（`x1, y1, x2, y2`）または `transform-origin` を厳密に再計算する。
  * アームの根元が、回転軸を表す円（白丸）の中心座標とピクセル単位で正確に一致するよう、オフセットやマージンのズレを解消する。
* **プロペラアニメーション（エアコン表示）**:
  * クルマのフロントグリルの両端に、SVGを用いたプロペラアイコンを配置する。
  * このアイコンは常に白色で常時表示される。エアコン（FanSpeed）の状態に応じて、CSSアニメーションによる回転速度を制御する（OFF時は停止状態とする）。
* **UIコンポーネントのレイアウト**:
  * WIPER制御セクションにはワイパーアイコン、DEFOGGER等には熱線アイコンなどを配置し、直感的な操作性を担保する。各セクションは論理的なグループ（環境設定、空調・ワイパー、窓制御、灯火類）に分割してレイアウトする。

### 3.2 シナリオ・ビューア (Scenario Viewer)
* **UI構造**: 事前定義されたスマートシーン・シナリオ（雨天時、サンキューハザード等）のロジックツリーを可視化するUI。
* **Read-Only表示**: ユーザーによるシナリオの直接編集や、シナリオ毎の個別RUN/STOP操作機能は持たない（全自動実行のみ）。自然言語の説明文と対応するOSDVI APIコードを上下に併記し、教育的な比較が容易なレイアウトとする。

### 3.3 シナリオ実行エンジン (Scenario Executor)
*   **ユーザー基本状態メモリ (User Memory State)**:
    *   シナリオ実行中にエッジ検出で状態を一時キャッシュする旧来の方式 (`PreRunStateCache`) を廃止し、永続的な `Internal.UserMemoryState` をストアに導入する。
    *   **START時の初期化**: イグニッションがSTOPからSTARTに遷移した際、`UserMemoryState` を初期化（ワイパーOFF、窓は現在値、防曇類OFF）する。
    *   **ユーザー操作の捕捉**: コンポーネントから直接呼び出される `setVss` アクションを監視し、対象アクチュエータ（窓、ワイパー、防曇類）のユーザー操作を検知した場合は `UserMemoryState` を即座に上書きする。
    *   **シナリオによる自動制御状態との分離**: 実行エンジンが `updateState` で発火させる自動制御アクションは `UserMemoryState` に影響を与えない。
    *   **RESTORE 処理**: シナリオ条件から脱した際の `RESTORE` アクションは、この `UserMemoryState` から値を読み出して復元する。
