# ソフトウェア要求仕様書：OSDVIスマートシナリオプロトタイプシステム

## 1. 用語定義
| 用語 | 読み | 説明 |
| :--- | :--- | :--- |
| スマートシーン | すまーとしーん | 車両のセンサ値（条件）に応じて、複数の車両機能を自動制御するロジック。 |
| イグニッション | いぐにっしょん | 車両の主電源（エンジンまたはモーター）の起動状態。START と STOP の状態を持つ。 |
| OSDVI | おーえすでぃーぶいあい | Open SDV Initiative。本システムが模倣すべき車載APIの標準規格（VSS準拠）。 |
| オーバーライド | おーばーらいど | 自動制御実行中に、ユーザーの手動操作（マニュアル操作）を優先して適用すること。 |

## 2. システム構成
本システムは、以下の2つのサブシステムから構成される。
1. **クルマシミュレータ**: ユーザーの操作や環境変化を受け付け、車両の状態を可視化する。本シミュレータ側に車両のイグニッション（START/STOP）機能を持つ。
2. **シナリオ・ビューア＆実行プログラム**: 100ms周期で条件を評価してシミュレータを制御する。

## 3. ソフトウェア要求一覧

### 3.1 機能要求：クルマシミュレータ（入力と表示）
* **イグニッション・スイッチ (SFR200)**: 
  * クルマシミュレータのセクションに、円形の「START / STOP」ボタンを配置する。**配置位置は「左寄せ（Left-aligned）」とする。**
  * プログラム起動時の初期状態は **STOP** とする。
  * **STOP状態の振る舞い**: 
    * ユーザーは窓・ワイパー・空調・灯火類のマニュアル操作が一切できない（スイッチ無効化）。
    * 状態は以下に強制固定される（ワイパー: OFF、窓: 現在位置維持、デフロスタ: OFF、デフォッガ: OFF、エアコン: OFF）。
* **環境・センサ設定 (SFR201)**: 
  * 雨量（0-100%）、後方車両距離（0-100m）を設定可能（これらは外部環境のためSTOP時も設定可能とする）。
* **手動操作入力 (SFR202) ※START状態時のみ操作可能**: 
  * **エアコン (CLIMATE)**: スイッチを「OFF / LO / MED / HI / AUTO」の5段階とする。
  * **窓スイッチ**: FL, FR, RL, RRの表記を「CL」「OP」と略記する。
    * 各窓の「CL (Close)」または「OP (Open)」ボタンを**押している間のみ**、窓がその方向に駆動する（離すと停止する）。駆動速度は全閉・全開に必要な時間（3秒）に準じる。
  * **ALL CLOSE / ALL OPEN ボタン**: 1回押すだけで目標値がセットされ、3秒間かけて自動で全開・全閉動作を行う。
  * **ターンシグナル**: 押すたびにON/OFFが切り替わるトグル式（RUN/STOP状態に応じてシナリオのトリガーとなる）。
* **状態表示 (SFR203)**: 
  * **アイコンの追加**: WIPERの左側にワイパーアイコン、DEFOGGER(REAR) ONの左側に熱線アイコン（横三本線）を表示する。
  * **エアコンのプロペラアニメーション**: クルマのグリルの両脇に配置する。OFFの時も常に表示（停止状態）させ、色を「白色」、サイズを大きめにする。
  * **窓の開度表示**: クルマの絵の下部に配置し、小数点以下を切り捨てた整数のみを表示する。**激しい雨天時でも文字が確実に読めるよう、背景は半透明ではなく「完全に不透明な黒（黒抜き）」とする。**
  * **中央ステータス表示**: 左右の窓のパーセント表示の間に、ウィンカ操作時は `LANE CHANGE`、ハザード点灯時は `♥THANKYU♥` と表示する。
  * **ワイパーの可視化**: **ワイパーのアーム（直線）の延長線上が、常に回転軸となる白丸の中心と幾何学的に完全に一致するように描画する。**静止状態（OFF）であってもズレが発生してはならない。
  * **雨天エフェクト**: 雨量の大小に関わらず雨粒の太さは一定（細い線）とし、量（密度）のみ変化させる。

### 3.2 機能要求：シナリオ・ビューア機能（教育用表示）
* シナリオは「自然言語」と「OSDVI API」を上下に並べて併記し、Read-Onlyで表示する。

### 3.3 必須搭載シナリオ
1. **雨天時スマートシーン**:
   * **(1) システム起動時の状態記憶**: 
     * 車両状態が「STOP」から「START」になった瞬間、その時点の状態を「ユーザーの基本状態」として内部に永続的に記憶する。
     * 記憶する初期値：ワイパー(OFF)、窓(全4席、現在の開閉状態)、デフォッガ(OFF)、デフロスタ(OFF)。
   * **(2) ユーザー操作による記憶の更新とシナリオのオーバーライド**: 
     * 「START」状態中はいつでもユーザーのボタン操作を最優先で受け付ける。雨天シナリオが自動制御を行っている最中であっても、ユーザーの操作が優先され、以後その機器に対するシナリオの自動動作は行われない。
     * 操作内容は「ユーザーの基本状態」として随時記憶される。
     * **ワイパー**: OFF/INT/LO/HI/AUTOボタン操作はRain Levelと無関係に独立して動作する。
     * **窓**: ALL CLOSE/ALL OPEN機能、および各窓（FL, FR, RL, RR）のCL/OPボタン（押している間だけ全閉/全開速度で動作）の操作を記憶する。
     * **防曇類**: デフォッガON/OFF、デフロスタON/OFFの操作を記憶する。
   * **(3) Rain Level が10%未満から、10%以上50%未満になった場合の自動設定**: 
     * ワイパーを「INT」、全窓を「全閉」、デフォッガ「ON」、デフロスタ「ON」に設定する。この状態は記憶しない。
   * **(4) Rain Level が10%未満以外から、10%以上50%未満になった場合の自動設定**: 
     * ワイパーを「INT」に変更する。窓、デフォッガ、デフロスタは「今の状態のまま」とする。この状態は記憶しない。
   * **(5) Rain Level が10%未満から、50%以上80%未満になった場合の自動設定**: 
     * ワイパーを「LO」、全窓を「全閉」、デフォッガ「ON」、デフロスタ「ON」に設定する。この状態は記憶しない。
   * **(6) Rain Level が10%未満以外から、50%以上80%未満になった場合の自動設定**: 
     * ワイパーを「LO」に変更する。窓、デフォッガ、デフロスタは「今の状態のまま」とする。この状態は記憶しない。
   * **(7) Rain Level が10%未満から、80%以上になった場合の自動設定**: 
     * ワイパーを「HI」、全窓を「全閉」、デフォッガ「ON」、デフロスタ「ON」に設定する。この状態は記憶しない。
   * **(8) Rain Level が10%未満以外から、80%以上になった場合の自動設定**: 
     * ワイパーを「HI」に変更する。窓、デフォッガ、デフロスタは「今の状態のまま」とする。この状態は記憶しない。
   * **(9) 条件未達時の状態復元 (RESTORE)**: 
     * Rain Levelが「10%以上」から「10%未満」になった瞬間に、自動制御を解除し、直前に記憶していた「ユーザーの基本状態」（ワイパー、窓、デフォッガ、デフロスタ）へと各機器を戻す。
2. **サンキューハザード**:
   * **(1) 発動条件**: 
     * 車両走行中（イグニッションがSTART状態）に、ユーザーがウインカー（左または右）を操作して点灯させた時。
   * **(2) 車線変更時の後方確認とハザード点滅**: 
     * ウインカー操作開始時の後方車両距離（Rear Distance）が30m以下であった場合、「サンキューハザード」のシーケンスが予約される。
     * 車線変更の完了を模擬するため、ウインカー操作開始から8秒間待機する。
     * 8秒経過後、自動でウインカーを消灯し、直ちにハザードランプ（左右のウインカ同時点滅）を点灯させる。
     * ハザード点灯から3秒経過後、自動でハザードランプを消灯し、一連のシーケンスを終了する。
     * ※後方距離が30mより大きい場合は、8秒経過後にウインカーが自動消灯するのみで、ハザードは点灯しない。

### 3.4 実行エンジン要求（イグニッション連動と高度なオーバーライド）
* **イグニッション状態連動**: 
  * 実行エンジンは100ms周期で動作し、イグニッション状態が「START」の間のみ、各スマートシーンシナリオの条件評価とアクション実行を行う。STOP状態ではすべての自動制御を即座に停止する。
* **ユーザー操作の優先権（オーバーライド）**: 
  * 自動制御実行中に、ユーザーが対象のアクチュエータ（窓、ワイパー、空調など）を手動で操作した場合、エンジンはその操作を検知し、該当アクチュエータに対するシナリオからの自動制御命令をブロックする。
  * このオーバーライド状態は、対象のシナリオ条件がリセットされる（例：雨が止み再び降り始めるなど）まで維持され、ユーザーの安全と意思決定を最優先する設計とする。
