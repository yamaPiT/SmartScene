### 1.1 目的
本書は、「OSDVIスマートシーンデモシステム V2.0」が『SW105_ソフトウェア要求仕様書』および『SW205_ソフトウェアアーキテクチャ設計書』に定義されたすべての機能・非機能要件を満たしていることを総合的に検証・報告するためのテストドキュメントである。

### 1.2 参照ドキュメント
* (SW105) ソフトウェア要求仕様書
* (SW205) ソフトウェアアーキテクチャ設計書
* OSDVI API 仕様書類（SW105の「7. その他・特記事項」に列挙の定義に基づく）

### 1.3 対象範囲とバージョン
* **対象ソフトウェア**: OSDVIスマートシーンデモシステム（SPAブラウザアプリケーション）
* **対象APIバージョン**: OSDVI 202603α

## 2. テスト環境とシステム構成

### 2.1 システム構成
* ブラウザ上で動作するSPA構成（外部ハードウェア連携なしのシミュレータ単体）。
* UIレイヤからZustand Storeを経由し、状態変更を3DモデルとUIに反映するFlux風アーキテクチャ。

### 2.2 テスト設備・ツール
* **テスト環境**: モダンWebブラウザ（Google Chrome, Edge等）および Local Node.js サーバ環境。
* **テスト手法**: UIからの手動操作、および開発者ツール等を用いたStoreの内部状態確認。

---

## 3. テスト方針

### 3.1 合否判定基準
以下の基準を満たした場合に本システムの「総合テスト完了」および「リリース適格」とする。
1. 第5章に記載された全テストケース（TC-GUI-001 〜 TC-NFR-002）の検査結果が「合格（✅ Pass）」となること。
2. その他テスト実行時に発覚した致命的な不具合（Critical/Blocker水準）が完全に解消されているか、フェイルセーフな回避策が講じられていること。

### 3.2 異常系・例外テスト方針
* **異常系・境界値**: UIまたは直接ストアに関数から限界値を超える値（例: 窓開度200%）を入力し、システムがクラッシュしないことを確認する（保護機能の動作確認）。
* **マニュアルオーバーライドと後勝ち原則**: 自動制御中における手動操作の割り込みと、シナリオ終了時の原状復帰（RESTORE）が正しく競合調停されるかを重点的に検証する。
